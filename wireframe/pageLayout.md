    화면 우측 상단~중단 : 일출 일몰 인디케이터 (시간이 지나면 점점 아래로 게이지가 내려감. 맨 위는 일출, 맨 아래는 일몰시간)
        만약 일몰 후~일출 전 시간일 경우 그래프 색상은 파란색이 되고, 맨 위의 일출을 향해 게이지가 올라감.
    화면 우측 하단 : 내 위치로 이동 버튼 (터치할 경우 GPS를 강제 활성화하여 내 위치를 탐색하고, 내 위치를 정중앙으로 하여 지도 이동)
    화면 우측 최하단 : 화면 확대 축소 컨트롤 바
    지도에서 고속도로 정보는 빼고, 국도와 지방도를 표시한다
    지도 화면은 사용자의 터치 입력에 따라 반응한다
        드래그 : 드래그에 맞추어 지도를 이동한다.
        핀치 줌 : 조작에 맞추어 지도를 확대/축소한다.
        터치 : 터치한 부분에 빨간색 위치 포인터가 표시되며, 그 아래에 빨간색의 '목적지'와 노란색의 '경유지' 플로팅 버튼이 나타난다.
            사용자의 현위치부터 터치한 지점까지가 반투명 원으로 표시되며, 원 바깥에 직선거리가 숫자로 나타난다. (예: 96km) 
            '목적지' 선택 시 빨간색 위치 포인터의 위치를 목적지로 설정한다.
                아래 [설정] 의 '목적지 자동추천'에서 '카페'를 선택한 경우, 사용자가 터치한 목적지 중심 반경 2km 이내의 카페를 찾아서 목적지를 자동 수정한다.
                    카페가 여러 군데일 경우 가까운 곳으로 선택한다
                    카페가 없을 경우 편의점을 찾아서 가까운 곳으로 선택한다.
                    반경 2km 이내에 카페와 편의점이 하나도 없는 경우 사용자가 터치한 목적지를 그대로 목적지로 설정한다.
                아래 [설정] 의 '목적지 자동추천'에서 '편의점'을 선택한 경우, 사용자가 터치한 목적지 중심 반경 2km 이내의 편의점을 찾아서 목적지를 자동 수정한다.
                    편의점이 여러 군데일 경우 가까운 곳으로 선택한다
                    편의점이 없을 경우 카페를 찾아서 가까운 곳으로 선택한다.
                    반경 2km 이내에 카페와 편의점이 하나도 없는 경우 사용자가 터치한 목적지를 그대로 목적지로 설정한다.
                아래 [설정] 의 '목적지 자동추천'에서 '추천안함'을 선택한 경우, 사용자가 터치한 목적지를 그대로 목적지로 설정한다.
            '경유지' 선택 시 빨간색 위치 포인터의 위치를 경유지로 설정하고, 위치 포인터를 경유지를 의미하는 노란색으로 변경한다.
            '목적지' 또는 '경유지' 플로팅 버튼 이외의 부분을 터치하면 기존 터치는 무시되고 새로 터치한 부분에 빨간색 위치 포인터가 표시되며, 그 아래에 빨간색의 '목적지'와 노란색의 '경유지' 플로팅 버튼이 나타난다.
            사용자가 화면을 터치한 시점에서 드래그 또는 핀치 줌을 할 경우, 위치 포인터와 플로팅 버튼과 반투명 원은 그대로 유지된다.
            화면상의 초록색 점으로 표시된 '추천 코스'는 화면을 확대하면 실제 해당 코스가 지도 위에 녹색 라인으로 표시된다.
                추천 코스를 터치하면 코스 설명 플로팅 화면이 뜬다.
                    코스 이름
                    코스 특징
                    코스 사진 (플로팅 안에서 좌우 슬라이드로 넘기도록 하고, 터치하면 추가 플로팅으로 큰 사진이 표시된다. 플로팅된 큰 사진은 핀치줌으로 확대축소, 터치하면 닫기 를 지원한다.) 
                    코스 등록자 정보 (프로필사진, 닉네임) : 터치하면 해당 사용자 프로필 카드가 추가 플로팅으로 뜬다. 프로필 카드는 프로필사진, 닉네임, 한줄소개, 받은 하트 수, 소셜미디어 아이콘, 바이크 정보가 표시된다.
                    하트 아이콘 : 터치하면 인스타그램처럼 하트가 표시된다. 기존 하트 숫자도 표시되며, 하트 숫자를 터치하면 누가 하트를 눌렀는지 프로필사진과 닉네임이 플로팅으로 표시된다. 하트의 갯수는 코스를 등록한 사람의 id에 누계되며, 프로필 플로팅에서 확인할 수 있다.
                    저장 아이콘 : 내가 선호하는 코스로 저장한다. 저장한 코스는 화면 상단의 '저장한 코스'에서 한 눈에 볼 수 있다.
                    경유지 버튼 : 경유지로 등록한다.
                    목적지 버튼 : 목적지로 등록한다.
    사용자가 목적지를 선택하면 화면 하단에 코스 선택 플로팅 버튼과 내비게이션 개시 슬라이드 버튼이 표시된다. 지도 상에는 아래 3가지 경로가 회색으로 표시된다.
        코스 선택 버튼: '시골길로 느긋하게' / '지방도로 여유롭게' / '국도로 빠르게' 로 경로탐색 옵션을 제시하며, 각각의 코스와 거리, 예상시간을 표시한다. 
            시골길로 느긋하게 : 지방도보다 등급이 낮은 도로를 최우선적으로 선택한다. 각각의 추천 코스마다 전체 거리의 10% 이내의 범위에서 거리가 늘어날 경우 추천 코스를 경유하도록 코스를 설정한다. 모든 이륜차 통행제한 도로는 제외한다.
            지방도로 여유롭게 : 최적경로 계산 시 지방도를 최우선적으로 선택하며, 국도는 지방도보다 선택 선호가 낮게 설정된다. 각각의 추천 코스마다 전체 거리의 5% 이내의 범위에서 거리가 늘어날 경우 추천 코스를 경유하도록 코스를 설정한다. 모든 이륜차 통행제한 도로는 제외한다.
            국도로 빠르게 : 최적경로 계산 시 국도를 최우선적으로 선택한다. '추천 코스'는 경로 계산의 고려 대상에서 제외된다. 모든 이륜차 통행제한 도로는 제외한다.
            여기에 맞추어 경로를 계산하고 주행거리와 주행시간을 표시한다.
        코스 선택 버튼을 터치하면 지도 위의 회색 경로 중 선택한 경로가 주황색으로 표시된다.
        최하단의 내비게이션 개시 슬라이드 버튼을 드래그하면 내비게이션 모드로 전환된다.

        핀치 줌 : 조작에 맞추어 지도를 확대/축소한다.
        터치 : 터치한 부분에 빨간색 위치 포인터가 표시되며, 그 아래에 초록색의 '출발지', 빨간색의 '목적지'와 노란색의 '경유지' 플로팅 버튼이 나타난다.
            '출발지' 선택 시 빨간색 위치 포인터의 위치를 출발지로 설정하고, 위치 포인터를 출발지를 의미하는 초록색으로 변경한다.
            '목적지' 선택 시 빨간색 위치 포인터의 위치를 목적지로 설정한다.
            '경유지' 선택 시 빨간색 위치 포인터의 위치를 경유지로 설정하고, 위치 포인터를 경유지를 의미하는 노란색으로 변경한다.
            '목적지' 또는 '경유지' 플로팅 버튼 이외의 부분을 터치하면 기존 터치는 무시되고 새로 터치한 부분에 빨간색 위치 포인터가 표시되며, 그 아래에 빨간색의 '목적지'와 노란색의 '경유지' 플로팅 버튼이 나타난다.
            사용자가 화면을 터치한 시점에서 드래그 또는 핀치 줌을 할 경우, 위치 포인터와 플로팅 버튼과 반투명 원은 그대로 유지된다.
    사용자가 출발지와 목적지를 선택하면 화면 하단에 코스 선택 플로팅 버튼이 표시된다. 지도 상에는 아래 3가지 경로가 회색으로 표시된다.
        코스 선택 버튼: '시골길로 느긋하게' / '지방도로 여유롭게' / '국도로 빠르게' 로 경로탐색 옵션을 제시하며, 각각의 코스와 거리, 예상시간을 표시한다. 
            시골길로 느긋하게 : 지방도보다 등급이 낮은 도로를 최우선적으로 선택한다. 각각의 추천 코스마다 전체 거리의 10% 이내의 범위에서 거리가 늘어날 경우 추천 코스를 경유하도록 코스를 설정한다. 모든 이륜차 통행제한 도로는 제외한다.
            지방도로 여유롭게 : 최적경로 계산 시 지방도를 최우선적으로 선택하며, 국도는 지방도보다 선택 선호가 낮게 설정된다. 각각의 추천 코스마다 전체 거리의 5% 이내의 범위에서 거리가 늘어날 경우 추천 코스를 경유하도록 코스를 설정한다. 모든 이륜차 통행제한 도로는 제외한다.
            국도로 빠르게 : 최적경로 계산 시 국도를 최우선적으로 선택한다. '추천 코스'는 경로 계산의 고려 대상에서 제외된다. 모든 이륜차 통행제한 도로는 제외한다.
            여기에 맞추어 경로를 계산하고 주행거리와 주행시간을 표시한다.
        코스 선택 버튼을 터치하면 지도 위의 회색 경로 중 선택한 경로가 주황색으로 표시되고, 등록정보 입력 플로팅 입력화면이 나타난다.
        [2단계]인 플로팅 입력화면은 다음 정보를 입력받는다.
            코스 이름 : 해당 코스를 나타내는 이름이 된다.
            코스 특징 : #숲길 #코너링 #자연 #시골마을 #전망 #위험 등 사용자 입력시 앞에 #이 표시되고 추천단어가 표시된다. 추천단어를 터치하면 바로 입력, 사용자가 직접 입력하면 입력을 받되 띄어쓰기 없이 단어로만 한정한다. 폭언 욕설 비속어 등은 차단한다.
            사진 등록 : 터치하면 스마트폰의 기본 갤러리(사진) 앱으로 사진을 선택하여 등록할 수 있게 한다.
                선택한 사진을 등록 시 이미지는 모두 flutter_image_compress 패키지를 사용해 가로 해상도 1080px, 압축률 Quality 70%로 하여 WebP 포맷으로 변환한다.
                이미지 스토리지 및 전송 아키텍처 (AWS S3 + CloudFront)
                    A. 업로드 프로세스 (Firebase Functions <-> AWS S3)
                        Presigned URL 생성: 클라이언트가 직접 AWS 키를 보유하지 않도록, Firebase Functions에서 AWS SDK를 사용하여 '보안 업로드용 임시 URL(Presigned URL)'을 생성해 클라이언트에 전달한다.
                        직접 업로드: 클라이언트는 변환된 WebP 파일을 해당 URL을 통해 AWS S3의 raw-uploads/ 버킷 경로로 직접 업로드한다.
                        파일 관리 규칙: 파일명은 userUid_timestamp.webp 형식을 권장하며, S3의 Metadata에 원본 파일 정보를 포함한다.
                    B. 이미지 서빙 및 캐싱 (AWS CloudFront CDN)
                        Origin 설정: AWS S3 버킷을 CloudFront의 오리진(Origin)으로 설정한다.
                        도메인 연결: assets.yurunavi.com과 같은 커스텀 도메인을 CloudFront에 연결하여 이미지 호출 경로를 통일한다. (예: https://assets.yurunavi.com/posts/image.webp)
                        캐싱 정책: 모든 이미지는 CloudFront 엣지 로케이션에 캐싱되어 S3 접근 횟수(GET Request)를 최소화한다. 브라우저 및 앱 캐시 유효 기간(TTL)을 길게 설정하여 반복 호출 시 데이터 전송 비용을 절감한다.
                    C. 비용 최적화 및 유지보수
                        S3 Lifecycle Policy: raw-uploads/ 경로의 파일 중 30일이 지난 데이터는 자동으로 S3 One Zone-IA 스토리지 클래스로 전환하여 저장 비용을 40% 이상 절감한다.
                        보안 (OAC): S3 버킷은 '퍼블릭 액세스 차단' 상태를 유지하고, 오직 CloudFront를 통해서만 이미지에 접근할 수 있도록 Origin Access Control(OAC)을 설정한다. 
            코스 이름, 코스 특징, 사진 등록 중 하단의 '취소'버튼을 터치하면 플로팅 화면이 사라지고 이전 화면으로 돌아간다. 이 때 사용자가 지정한 출발지, 경유지, 목적지는 그대로 유지된다. (코스 선택 버튼 터치 바로 전 단계로 이동)

             코스 이름, 코스 특징, 사진 등록 후 하단의 '등록' 버튼을 터치하면 기존 등록 코스 데이터와 비교 검증하여, 70% 이상 겹치지 않는 경우 코스를 등록한다.
